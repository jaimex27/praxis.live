video:capture

Video capture support for using webcams, etc. Generally you should use a generic device URI (capture://0, capture://1, etc.) which will work cross-platform. The first device is index 0 and so on.

If your input device supports multiple capture sizes or framerates, it is possible to specify these using the URI query string - eg. capture://0?width=640&height=480&framerate=15/1. Be aware that the device will fail to open if the requested settings are not supported.

A capture device will not start automatically. To get it to start when the video root is started, use a core:start-trigger attached to the play port.

Ports

ID Type Direction Values Description
out video out Video output.
play control in any Start capture device.
stop control in any Stop capture device.

Controls

ID Type Input Output Default Description
device property resource {} Capture device - see details above.
